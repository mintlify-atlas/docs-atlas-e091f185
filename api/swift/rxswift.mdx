---
title: "KMPNativeCoroutinesRxSwift"
description: "RxSwift API reference for the KMPNativeCoroutinesRxSwift package. Wraps Kotlin flows and suspend functions as RxSwift Observables and Singles."
---

The `KMPNativeCoroutinesRxSwift` package provides [RxSwift](https://github.com/ReactiveX/RxSwift) integration for KMP-NativeCoroutines. It converts `NativeFlow` and `NativeSuspend` values into RxSwift `Observable`s and `Single`s with full cancellation support.

**Platform requirements:** Swift 5.0, iOS 9.0+, macOS 10.9+, tvOS 9.0+, watchOS 3.0+

<Note>
  All observables and singles returned by this package are **deferred**. Every new subscription triggers a fresh collection of the flow or a new execution of the suspend function. Disposing the subscription cancels the underlying Kotlin coroutine.
</Note>

---

## createObservable(for:)

Creates an `Observable` for a `NativeFlow` or for a `NativeSuspend` that returns a `NativeFlow`. Each subscription starts a new, independent flow collection.

### NativeFlow overload

```swift
public func createObservable<Output, Failure: Error, Unit>(
    for nativeFlow: @escaping NativeFlow<Output, Failure, Unit>
) -> Observable<Output>
```

<ParamField path="nativeFlow" type="NativeFlow<Output, Failure, Unit>" required>
  The native flow to collect. Generated by the Gradle plugin from a `@NativeCoroutines`-annotated Kotlin `Flow` property or function.
</ParamField>

**Returns:** An `Observable<Output>` that emits each value from the flow, completes when the flow finishes, or errors if the flow throws.

```swift
import KMPNativeCoroutinesRxSwift

let observable = createObservable(for: clock.time)

let disposable = observable.subscribe(
    onNext: { value in
        print("Received value: \(value)")
    },
    onError: { error in
        print("Received error: \(error)")
    },
    onCompleted: {
        print("Observable completed")
    },
    onDisposed: {
        print("Observable disposed")
    }
)

// Cancel the flow collection
disposable.dispose()
```

### Unit-emitting flow overload

Use this overload when the flow emits `Unit` values (i.e., `Void` in Swift).

```swift
public func createObservable<Unit, Failure: Error>(
    for nativeFlow: @escaping NativeFlow<Unit, Failure, Unit>
) -> Observable<Void>
```

<ParamField path="nativeFlow" type="NativeFlow<Unit, Failure, Unit>" required>
  The native flow to collect. The flow must emit `Unit` on the Kotlin side.
</ParamField>

**Returns:** An `Observable<Void>`.

### NativeSuspend returning NativeFlow overload

Creates an `Observable` for a suspend function that returns a `NativeFlow`. This is a convenience overload that first awaits the suspend function as a `Single`, then collects the resulting flow as an `Observable`.

```swift
public func createObservable<Output, Failure: Error, Unit>(
    for nativeSuspend: @escaping NativeSuspend<NativeFlow<Output, Failure, Unit>, Failure, Unit>
) -> Observable<Output>
```

<ParamField path="nativeSuspend" type="NativeSuspend<NativeFlow<Output, Failure, Unit>, Failure, Unit>" required>
  The native suspend function that returns a `NativeFlow`.
</ParamField>

**Returns:** An `Observable<Output>` that publishes the values from the flow returned by the suspend function.

```swift
import KMPNativeCoroutinesRxSwift

// suspend function that returns a Flow
let observable = createObservable(for: randomLettersGenerator.getRandomLettersFlow())

let disposable = observable.subscribe(onNext: { letters in
    print("Received: \(letters)")
}, onError: { error in
    print("Error: \(error)")
})
```

### Unit-emitting NativeSuspend returning NativeFlow overload

```swift
public func createObservable<Unit, Failure: Error>(
    for nativeSuspend: @escaping NativeSuspend<NativeFlow<Unit, Failure, Unit>, Failure, Unit>
) -> Observable<Void>
```

<ParamField path="nativeSuspend" type="NativeSuspend<NativeFlow<Unit, Failure, Unit>, Failure, Unit>" required>
  The native suspend function that returns a `NativeFlow` of `Unit`.
</ParamField>

**Returns:** An `Observable<Void>`.

## createSingle(for:)

Creates a `Single` for a `NativeSuspend`. The single emits exactly one value and completes, or fails with an error. Disposing the subscription cancels the underlying Kotlin coroutine.

### Main overload

```swift
public func createSingle<Result, Failure: Error, Unit>(
    for nativeSuspend: @escaping NativeSuspend<Result, Failure, Unit>
) -> Single<Result>
```

<ParamField path="nativeSuspend" type="NativeSuspend<Result, Failure, Unit>" required>
  The native suspend function to await. Generated by the Gradle plugin from a `@NativeCoroutines`-annotated Kotlin suspend function.
</ParamField>

**Returns:** A `Single<Result>` that either succeeds with the result value or fails with an error.

```swift
import KMPNativeCoroutinesRxSwift

let single = createSingle(for: randomLettersGenerator.getRandomLetters())

let disposable = single.subscribe(
    onSuccess: { value in
        print("Received value: \(value)")
    },
    onFailure: { error in
        print("Received error: \(error)")
    },
    onDisposed: {
        print("Single disposed")
    }
)

// Cancel the underlying coroutine
disposable.dispose()
```

### Unit-returning overload

```swift
public func createSingle<Unit, Failure: Error>(
    for nativeSuspend: @escaping NativeSuspend<Unit, Failure, Unit>
) -> Single<Void>
```

<ParamField path="nativeSuspend" type="NativeSuspend<Unit, Failure, Unit>" required>
  The native suspend function to await. The function must return `Unit` on the Kotlin side.
</ParamField>

**Returns:** A `Single<Void>`.

### Swift export compat overload

<Warning>
  This overload is deprecated. Migrate away from it once you have fully moved to Swift export.
</Warning>

```swift
@available(*, deprecated, message: "Kotlin Coroutines are supported by Swift export")
@available(iOS 13.0, macOS 10.15, tvOS 13.0, watchOS 6.0, *)
public func createSingle<Result>(
    for operation: @escaping () async throws -> Result
) -> Single<Result>
```

This overload wraps an arbitrary `async throws` closure in a `Single` backed by a Swift `Task`. It exists for source compatibility during migration to Swift export.

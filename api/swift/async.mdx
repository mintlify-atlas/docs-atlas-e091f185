title: "KMPNativeCoroutinesAsync"
description: "Swift Concurrency API reference for the KMPNativeCoroutinesAsync package. Wraps Kotlin coroutines as async functions and AsyncSequences."
---

The `KMPNativeCoroutinesAsync` package provides Swift Concurrency integration for KMP-NativeCoroutines. It converts `NativeSuspend` and `NativeFlow` values into native Swift async functions and `AsyncSequence`s, with full cancellation support.

**Platform requirements:** Swift 5.5, iOS 13.0+, macOS 10.15+, tvOS 13.0+, watchOS 6.0+

---

## asyncFunction(for:)

Wraps a `NativeSuspend` in an async function that can be awaited and supports cancellation.

### Main overload

```swift
public func asyncFunction<Result, Failure: Error, Unit>(
    for nativeSuspend: @escaping NativeSuspend<Result, Failure, Unit>
) async throws -> Result
```

<ParamField path="nativeSuspend" type="NativeSuspend<Result, Failure, Unit>" required>
  The native suspend function to await. Generated by the Gradle plugin from a `@NativeCoroutines`-annotated Kotlin suspend function.
</ParamField>

**Returns:** The result value of type `Result` produced by the suspend function.

**Throws:** Any `Failure` error thrown by the suspend function, or a `CancellationError` if the enclosing Swift `Task` is cancelled.

```swift
import KMPNativeCoroutinesAsync

let handle = Task {
    do {
        let letters = try await asyncFunction(for: randomLettersGenerator.getRandomLetters())
        print("Got random letters: \(letters)")
    } catch {
        print("Failed with error: \(error)")
    }
}

// Cancel the underlying Kotlin coroutine by cancelling the Task
handle.cancel()
```

### Unit-returning overload

Use this overload when the suspend function returns `Unit` (i.e., `Void` in Swift).

```swift
public func asyncFunction<Unit, Failure: Error>(
    for nativeSuspend: @escaping NativeSuspend<Unit, Failure, Unit>
) async throws -> Void
```

<ParamField path="nativeSuspend" type="NativeSuspend<Unit, Failure, Unit>" required>
  The native suspend function to await. The function must return `Unit` on the Kotlin side.
</ParamField>

**Returns:** `Void`.

**Throws:** Any `Failure` error thrown by the suspend function, or a `CancellationError` if the enclosing Swift `Task` is cancelled.

```swift
import KMPNativeCoroutinesAsync

try await asyncFunction(for: myRepository.syncData())
```

### Swift export compat overload

<Warning>
  This overload is deprecated. Remove it once you have fully migrated to Swift export.
</Warning>

```swift
@available(*, deprecated, message: "Kotlin Coroutines are supported by Swift export")
public func asyncFunction<Result>(for result: Result) async -> Result
```

This is a no-op function that returns its argument unchanged. It exists to provide source compatibility during migration to Swift export, where Kotlin suspend functions are called directly without wrapping.

## asyncResult(for:)

Awaits a `NativeSuspend` and returns a Swift `Result` instead of throwing. Useful when you want to handle success and failure in a single expression.

### Main overload

```swift
public func asyncResult<Output, Failure: Error, Unit>(
    for nativeSuspend: @escaping NativeSuspend<Output, Failure, Unit>
) async -> Result<Output, Error>
```

<ParamField path="nativeSuspend" type="NativeSuspend<Output, Failure, Unit>" required>
  The native suspend function to await.
</ParamField>

**Returns:** `.success(Output)` if the suspend function completes successfully, or `.failure(Error)` if it throws or is cancelled.

```swift
import KMPNativeCoroutinesAsync

let result = await asyncResult(for: randomLettersGenerator.getRandomLetters())
switch result {
case .success(let letters):
    print("Got random letters: \(letters)")
case .failure(let error):
    print("Failed with error: \(error)")
}
```

### Unit-returning overload

```swift
public func asyncResult<Unit, Failure: Error>(
    for nativeSuspend: @escaping NativeSuspend<Unit, Failure, Unit>
) async -> Result<Void, Error>
```

<ParamField path="nativeSuspend" type="NativeSuspend<Unit, Failure, Unit>" required>
  The native suspend function to await. The function must return `Unit` on the Kotlin side.
</ParamField>

**Returns:** `.success(())` or `.failure(Error)`.

### Swift export compat overload

<Warning>
  This overload is deprecated. Replace it with a `do`-`catch` once you have fully migrated to Swift export.
</Warning>

```swift
@available(*, deprecated, message: "Kotlin Coroutines are supported by Swift export")
public func asyncResult<Output>(
    for output: @autoclosure () async throws -> Output
) async -> Result<Output, Error>
```

## asyncError(for:)

Awaits a `NativeSuspend` that returns `Unit` and returns the optional error. Useful for fire-and-forget suspend functions where you only care whether they succeeded or failed.

```swift
public func asyncError<Unit, Failure: Error>(
    for nativeSuspend: @escaping NativeSuspend<Unit, Failure, Unit>
) async -> Error?
```

<ParamField path="nativeSuspend" type="NativeSuspend<Unit, Failure, Unit>" required>
  The native suspend function to await. The function must return `Unit` on the Kotlin side.
</ParamField>

**Returns:** The `Error` thrown by the suspend function, or `nil` if it completed successfully or was cancelled without error.

```swift
import KMPNativeCoroutinesAsync

if let error = await asyncError(for: integrationTests.returnUnit()) {
    print("Failed with error: \(error)")
}
```

## asyncSequence(for:)

Wraps a `NativeFlow` in a `NativeFlowAsyncSequence` that can be iterated with `for`-`await`-`in`. Cancelling the enclosing `Task` cancels the underlying Kotlin flow collection.

```swift
public func asyncSequence<Output, Failure: Error, Unit>(
    for nativeFlow: @escaping NativeFlow<Output, Failure, Unit>
) -> NativeFlowAsyncSequence<Output, Error, Unit>
```

<ParamField path="nativeFlow" type="NativeFlow<Output, Failure, Unit>" required>
  The native flow to collect. Generated by the Gradle plugin from a `@NativeCoroutines`-annotated Kotlin `Flow` property or function.
</ParamField>

**Returns:** A `NativeFlowAsyncSequence` that yields each emitted value in order.

```swift
import KMPNativeCoroutinesAsync

let handle = Task {
    do {
        let sequence = asyncSequence(for: clock.time)
        for try await time in sequence {
            print("Current time: \(time)")
        }
    } catch {
        print("Flow failed with error: \(error)")
    }
}

// Cancel the underlying Kotlin flow by cancelling the Task
handle.cancel()
```

## NativeFlowAsyncSequence

An `AsyncSequence` that collects values from a Kotlin `NativeFlow`. Returned by `asyncSequence(for:)`.

```swift
public struct NativeFlowAsyncSequence<Output, Failure: Error, Unit>: AsyncSequence {
    public typealias Element = Output
}
```

| Generic parameter | Description |
|---|---|
| `Output` | The type of each element emitted by the flow. |
| `Failure` | The error type. Always `Error` when constructed via `asyncSequence(for:)`. |
| `Unit` | The Kotlin `Unit` type parameter used internally. |

`NativeFlowAsyncSequence` conforms to `AsyncSequence` and produces an `Iterator` via `makeAsyncIterator()`. The iterator calls back into the Kotlin flow using the native callbacks, suspending between elements and propagating errors or task cancellation.

<Note>
  Each call to `makeAsyncIterator()` â€” including each `for`-`await`-`in` loop â€” starts a new, independent collection of the underlying Kotlin flow.
</Note>
